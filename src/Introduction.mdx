[@thewronghand](https://github.com/thewronghand)의 개인 프로젝트용 UI 라이브러리입니다.
사이드프로젝트에서 `npm install` 후 슥슥 꺼내 씁니다.

---

## 설치

```bash
npm install @thewrong/ui
```

```tsx
// 1. 스타일 한 줄 (Tailwind 설치 불필요 — CSS가 번들에 자체포함)
import "@thewrong/ui/styles.css";

// 2. 컴포넌트 import
import { Button, Modal, PaginatedTable, SearchBox } from "@thewrong/ui";
```

무거운 의존(motion, floating-ui, dnd-kit, react-virtual, date-fns, react-hot-toast)은
**optional peerDependency**라, 그 기능을 쓰는 컴포넌트만 해당 패키지를 함께 설치하면 됩니다.

## 테마 / 스타일 커스터마이징

색을 바꾸는 길은 두 가지이고, **목적에 따라 고른다.**

### 전역 색 교체 — CSS Variable 토큰 오버라이드 (대부분 이걸로 충분)

색은 2계층 토큰입니다 — 원시 토큰(oklch)을 시맨틱 토큰이 `var()`로 참조합니다.
소비자가 `:root`에서 CSS Variable을 덮어쓰면 **그 토큰을 쓰는 모든 컴포넌트의 색이 한 번에** 바뀝니다.
앱 전체를 자기 디자인 톤에 맞추려면 이 방법을 쓰세요(컴포넌트 코드를 만질 필요 없음).

```css
:root {
  /* 원시 토큰 — 브랜드 색 계열 교체 */
  --color-primary-500: oklch(0.6 0.2 260);

  /* 시맨틱 토큰 — 배경·텍스트·보더를 직접 지정 (앱 톤 맞추기) */
  --color-bg-white: oklch(0.16 0.01 260); /* 카드/시트 배경 */
  --color-text-primary: oklch(0.95 0 0); /* 본문 텍스트 */
  --color-border-tertiary: oklch(0.3 0.01 260); /* 옅은 구분선 */
}
.dark {
  /* 다크 토큰도 같은 이름으로 오버라이드 */
}
```

시맨틱 토큰 계열: `bg-*`(base-primary/secondary/tertiary·white·card·disabled) · `text-*`(primary/secondary/tertiary/disabled/inverse/danger) · `border-*` · `icon-*`. 전체 목록은 `src/styles/theme.css` 참고.

### 인스턴스별 스타일 — `className` prop

"같은 앱 안에서 **이 컴포넌트 하나만** 폭·여백·배경을 다르게"는 토큰으로는 안 됩니다.
많은 컴포넌트가 `className`을 받아 루트에 병합하므로, 인스턴스 단위로 덮으세요.

```tsx
<Toolbar className="bg-slate-900" ... />
<Table className="rounded-xl" ... />
```

> floating·고정 좌표 컴포넌트(Tooltip·DatePicker 등)는 위치가 깨질 수 있어 `className`을 열지 않습니다.
> 그 경우에도 **색은 위의 토큰 오버라이드로** 바뀝니다.

- **폼** — Button · Input(+PasswordInput) · Textarea · Select · MultiSelect · Switch · Checkbox · Badge
- **오버레이** — Modal(+SubView/Standard) · Drawer · Popover · Tooltip · InfoTooltip · Toast · ActionToast
- **날짜** — DatePicker · DateRangePicker · MonthPicker · DateInput

## 사용 가이드 — 언제 무엇을 쓰나

비슷한 컴포넌트가 여럿일 때 무엇을 골라야 하는지, 어떻게 조합하는지에 대한 결정 가이드입니다.
(AI 코드 어시스턴트에게도 이 문서를 그대로 컨텍스트로 주면 적절한 선택을 합니다.)

### 오버레이 — 무엇으로 띄울까

| 상황 | 선택 | 이유 |
| --- | --- | --- |
| 폼·상세를 화면 중앙에 띄움 | **Modal** | 데스크탑 중앙 카드, 모바일 바텀시트로 자동 분기 |
| 표준 폼(확인/취소 버튼) | **StandardModal** | 폼 80% 케이스. `primaryAction`/`secondaryAction`으로 footer 자동 생성. 자유도가 더 필요하면 base `Modal`로 내려가 `footer`를 직접 그림 |
| 모달 안에서 한 단계 더 들어감 | **ModalSubView** | 우측에서 슬라이드 인하는 푸시 네비. **모달 위에 또 모달을 쌓지 말 것** — 레이어드 모달 대신 이걸로 푼다 |
| 옆에서 밀려나오는 패널(필터·상세) | **Drawer** | 화면 가장자리 고정. 모바일은 바텀시트 |
| 사용자에게 즉시 결정을 요구(컨펌) | **ActionToast** | 자동 dismiss 없음, 한 번에 하나. **모달 위에서도 띄울 수 있어** "모달 안에서 추가 확인/모드 전환 컨펌"에 쓴다 |
| 단순 알림(저장됨, 실패함) | **Toast** | 자동 사라짐. `ToastProvider`를 앱 루트에 한 번 두고 `toast()` 호출 |

> **모달 위에 모달을 쌓지 않는다.** 더 깊은 뷰는 `ModalSubView`(푸시 네비), 재확인은
> `ActionToast`로 푼다. ESC·백드롭은 `overlayStack`에서 자기가 top일 때만 반응하므로
> 레이어가 겹쳐도 가장 위 것만 닫힌다.

### 모달 — 알아둘 정책

- **너비는 고정한다** — 동적 너비는 중앙정렬 `translateX(-50%)` 기준점과 함께 움직여 한쪽으로
  밀린다. 콘텐츠 폭이 인터랙션으로 변하면 `widthPx`로 고정할 것.
- **높이가 변하는 본문은 `AnimatedHeight`로 감싼다** — 토글·조건부 영역이 있으면 시트 높이가
  부드럽게 이행된다. 미적용 시 instant jump로 시선이 흔들린다.
- **ModalSubView 콘텐츠 높이 ≤ 부모 모달 본문 높이** — 서브뷰는 부모 시트를 그대로 덮으므로,
  작은 모달에 아주 긴 서브뷰를 넣으면 높이가 어긋나 보인다. 긴 목록은 부모를 키우거나 별도 페이지로.

### 셀렉트 — 어느 변형을 쓸까

| 상황 | 선택 |
| --- | --- |
| 고정된 옵션 중 하나 | **Select** |
| 옵션이 많아 타이핑으로 좁히고 싶음(옵션은 이미 다 가짐) | **Select** `searchable`(클라이언트 필터) |
| 옵션을 서버에서 검색해 받아옴 | **Select** `searchable` + 서버 검색(호출부가 selected snapshot 보존해 깜빡임 방지) |
| 여러 개 선택 | **MultiSelect** (값 `string[]`, 트리거 폭 넘으면 'N개 선택'으로 자동 폴백) |

### 도움말·작은 오버레이 — Tooltip / InfoTooltip / Popover

| 상황 | 선택 |
| --- | --- |
| hover/focus로 짧은 텍스트 힌트 | **Tooltip** (content는 string) |
| 물음표 아이콘에 붙는 도움말(강조·줄바꿈 필요) | **InfoTooltip** (content는 ReactNode) |
| 클릭으로 여닫는 자유 콘텐츠(헤더 드롭다운 등) | **Popover** (content는 ReactNode) |

### 테이블 — Table / PaginatedTable / AccordionTable

| 상황 | 선택 |
| --- | --- |
| 대부분의 목록 페이지 | **PaginatedTable** — 페이지네이션 + 페이지 사이즈 셀렉터 내장 |
| 데이터를 한 번에 받아 전부 표시 | **Table** — 페이지네이션 없음. `totalElements`를 주면 '총 N건' 자동 표기 |
| 행을 펼쳐 상세를 보여줌(비동기 fetch 가능) | **AccordionTable** — single/multiple 펼침 모드 |
| 체크박스만 단독 | **TableCheckbox** (indeterminate 지원) |

> **이중 박스 주의** — Table/PaginatedTable은 자체 padding·border가 있어, 일반 Section의
> `p-4` 박스에 넣으면 박스가 겹쳐 뚱뚱해진다. `bg-white rounded-lg overflow-hidden` wrapper로만 감쌀 것.

### 검색 바 — SearchBox

선언적 필드 배열로 검색 바를 구성한다. 필드 타입: `text`(단일 키워드) · `select`(단일) ·
`multiSelect`(다중) · `dateRange`(from/to + 프리셋 + 캘린더). 비동기 fetch 중에는 `isSubmitting`을
주어 버튼을 spinner + disabled로 만들어 중복 요청을 막는다. 모바일은 자동으로 접힘 토글이 생긴다.

### 공통 동작

- **모바일 분기** — Modal·Drawer·DatePicker류는 `sm` 미만에서 화면 하단 바텀시트로 자동 전환되고,
  상단 핸들 바를 잡고 아래로 끌어 닫을 수 있다(데스크탑은 드래그 비활성).
- **사이즈 토큰** — Input·Select·Textarea·Button은 `mini`/`small`/`medium`/`large` 4단계를
  공유 토큰(height·rounded·padding)으로 정합한다.

## 설계 원칙

기술적 한계나 의도적 결정으로 **정책으로 풀어낸 제약**은 "제약 + 이유 + 대안"으로 명시해 둡니다.
예) 모달 너비는 고정 — 동적 너비는 중앙정렬 transform과 충돌해 모션이 부자연스럽다. 폭이 변하면 `widthPx`로 고정할 것.
